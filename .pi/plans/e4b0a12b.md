## Investigation Complete

### Root Causes Identified

**Issue #1: DNS Query Domain = Process Name**
- **Root Cause**: eBPF probe uses `bpf_get_current_comm()` instead of extracting actual DNS query from packet
- **Location**: `zig/ebpf/dns_query.c:25`
- **Fix**: Remove `bpf_get_current_comm()` call, leave qname empty
- **Impact**: 1 line change

**Issue #2: File Open Events Have Null file_path**
- **Root Cause**: Two-level problem:
  1. Enrichment may fail to extract path from payload (payload parsing issue)
  2. Detection event has no fallback to raw payload for file events
- **Locations**:
  - Primary: `crates/agent-core/src/lifecycle/detection_event.rs:49-53`
  - Secondary: `crates/platform-linux/src/lib.rs:394-396`
- **Fix**: Add `.or_else()` fallback to parse raw payload
- **Impact**: ~18 lines for primary fix, ~10 lines for secondary
